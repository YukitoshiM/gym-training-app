# アルファ版スコープ定義

作成日: 2026-06-20  
対象: iPhoneアプリ アルファ版  
基準資料: `gym_training_app_design_mvp.md`, `gym_training_app_user_stories_mvp.md`

## 1. アルファ版の目的

アルファ版は、MVP全体を完成させる前に、トレーニング記録アプリの中核体験を実機で確認するための検証版である。

検証したい中核体験は以下とする。

1. 種目を選んで計画を作れる
2. 計画からワークアウトを開始できる
3. セットごとの目標重量・目標回数に対して実績回数を入力できる
4. セットごとの達成/未達と差分が分かる
5. ワークアウト完了後に履歴として見返せる

アルファ版では、見た目の完成度よりも「ジム中に迷わず記録できるか」「計画との差分が気持ちよく分かるか」を優先する。

## 2. アルファ版の基本方針

- ユーザーストーリーを単位にスコープを管理する。
- 1つの縦切り体験を優先し、周辺機能は後回しにする。
- データはローカル保存を基本とする。
- クラウド同期、課金、AI提案、Apple連携は含めない。
- UIはSwiftUI標準コンポーネント中心でよいが、片手入力しやすい記録画面は初期から意識する。

## 5. アルファ版のデータ保存

アルファ版では、ローカル保存のみで開始する。

推奨:

- SwiftDataを使用する
- 保存対象は種目、計画、ワークアウト履歴
- プリセット種目はアプリ内の静的データとして持つ
- CloudKit同期は後続で追加できるよう、モデルには `id`, `createdAt`, `updatedAt` を持たせる

## 6. 最初の実装順

### Milestone A1: アプリの土台

- タブ構成を作る
- SwiftDataまたは一時的なインメモリストアを用意する
- プリセット種目を表示する

完了条件:

- アプリを起動して、ホーム・計画・記録・履歴タブを移動できる
- 種目一覧を表示できる

### Milestone A2: 計画作成

- 計画一覧を作る
- 計画作成画面を作る
- 種目追加、セット目標入力、保存を実装する

完了条件:

- 「胸の日」などの計画を作成できる
- 計画に複数種目と複数セットを登録できる
- アプリ再起動後も計画が残る

### Milestone A3: ワークアウト記録

- 計画からワークアウトを開始する
- セット実績を入力する
- 完了チェック、セット追加・削除、種目スキップを実装する

完了条件:

- 計画を選んで記録を開始できる
- セットごとに実績重量・実績回数を保存できる
- 目標との差分と達成/未達が見える

### Milestone A4: 完了・履歴

- ワークアウト完了画面を作る
- 種目達成率、全体達成率、総ボリュームを計算する
- 履歴一覧と履歴詳細を作る

完了条件:

- ワークアウト完了後に履歴へ残る
- 履歴詳細でセット内容を確認できる
- 総ボリュームと達成率が表示される

### Milestone A5: 使い勝手改善

- 目標値コピー
- 前回記録表示
- 削除・終了確認
- 簡易休憩タイマー

完了条件:

- ジム中の入力ステップが減っている
- 誤操作で記録を失いにくい

## 7. アルファ版の受け入れ基準

アルファ版は、以下を満たしたら「触れる版」として成立とする。

- プリセット種目から計画を作成できる
- 計画にセットごとの目標重量・目標回数を設定できる
- 計画からワークアウトを開始できる
- セットごとに実績重量・実績回数を入力できる
- 実績回数と目標回数の差分が自動表示される
- セットごとの達成/未達が分かる
- ワークアウト完了時に種目達成率・全体達成率・総ボリュームが表示される
- 完了したワークアウトを履歴一覧から見返せる
- アプリ再起動後も計画と履歴が残る

## 8. アルファ版で判断したいこと

| 検証観点 | 確認したいこと |
|---|---|
| 計画作成 | セットごとの目標入力が面倒すぎないか |
| 記録画面 | ジム中に片手で入力できるか |
| 差分表示 | 達成/未達が一瞬で理解できるか |
| 履歴 | 次回トレーニングの参考になる情報が残っているか |
| 継続性 | 毎回使う気になる入力ステップ数か |
