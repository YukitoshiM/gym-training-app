# AIボディメイクマネージャー プロダクト方針

作成日: 2026-07-04  
対象: iPhoneアプリ / プロトタイプ / MVP

## 1. コンセプト

本アプリは、ダイエット、筋肥大、健康維持、体型改善、競技力向上をまとめて管理するAIボディメイクマネージャーである。

単なる記録アプリではなく、体重、腹囲、体型写真、食事、筋トレ記録をつなげて、次に何を変えるべきかを提案する改善管理アプリとして設計する。

中心価値は以下。

> 記録するアプリではなく、改善を管理するアプリ。

ユーザーが解決したいこと:

- 体重を減らしたい
- 腹囲を減らしたい
- 筋肉を増やしたい
- 見た目の変化を確認したい
- 食事を楽に記録したい
- 筋トレの計画と実績差を見たい
- 停滞した時に原因を知りたい
- 次に何をすればいいか知りたい

## 2. ターゲット

### メインターゲット

一般ユーザーのボディメイク。

- ダイエット中の人
- 筋トレ初心者から中級者
- 健康維持したい人
- 腹囲や体型を管理したい人
- 食事記録が面倒で続かない人

### サブターゲット

アスリート、競技者、部活、趣味スポーツ層。

初期から本格アスリート向けにしすぎない。最初は「競技力向上モード」として、疲労度、練習量、体重変動、食事量、パフォーマンスメモを見られる程度に留める。

## 3. 目的モード

初回登録時に目的を選ばせる。

| 目的 | 主に見る指標 |
|---|---|
| ダイエット | 体重、腹囲、摂取カロリー、脂質、歩数、体型写真、停滞日数 |
| 筋肥大 | 体重、たんぱく質、トレーニングボリューム、種目別重量、種目別回数、体型写真、部位別進捗 |
| 健康維持 | 体重、腹囲、食事バランス、運動頻度、睡眠、疲労度 |
| 体型改善 | 体型写真、腹囲、肩幅感、腹部の見た目、脇腹の変化、姿勢、筋トレ部位バランス |
| 競技力向上 | 練習量、筋トレ量、体重、食事量、疲労度、睡眠、パフォーマンスメモ |

## 4. 入力・分析・出力

### 入力

- 体重
- 腹囲
- 体型写真
- 食事写真
- 筋トレ記録
- 睡眠
- 疲労度
- 目標

### 分析

- 前回との差
- 目標との差
- 停滞の原因
- 食事と体型変化の関係
- 筋トレ量と身体変化の関係

### 出力

- 今日の注意点
- 次回トレーニングの調整
- 食事の改善案
- 減量、増量、維持の方針
- 写真ベースの見た目変化コメント

## 5. 主要機能

### 5.1 ダッシュボード

ホーム画面に以下を表示する。

- 今日の体重
- 今日の腹囲
- 今日の食事記録状況
- 今日の筋トレ予定
- 直近7日間の変化
- AIからの一言
- 次にやること

AIコメント例:

> 腹囲は順調に減っています。体重は横ばいですが、写真では腹部が少し引き締まっています。今日の食事は脂質を40g以内に抑えるとよさそうです。

### 5.2 身体測定記録

必須項目:

- 体重
- 腹囲

任意項目:

- 胸囲
- 腕周り
- 太もも
- ふくらはぎ
- 肩幅
- 体脂肪率
- メモ

グラフ:

- 体重推移
- 腹囲推移
- 体重と腹囲の比較
- 週平均
- 月平均

### 5.3 体型写真管理

撮影パターン:

- 正面
- 横
- 背面
- 腹部アップ

AIで見る内容:

- 腹部の変化
- 脇腹の変化
- 肩周りの変化
- 胸周りの変化
- 姿勢
- 前回との差
- 見た目スコア

注意点:

- 写真だけで体脂肪率を断定しない。
- 出す場合は「推定」「参考値」とする。
- 重要なのは絶対値ではなく、同じ条件で撮った写真の変化。

AIコメント例:

> 前回より腹部の張り出しが少し減っています。体重は大きく変わっていませんが、見た目では腹囲まわりに改善が見えます。同じ光・同じ姿勢で撮ると、もっと比較しやすくなります。

### 5.4 食事管理

食事は写真で記録できるようにする。

入力:

- 食事写真
- 食事名
- 時刻
- メモ
- 手動補正

AI下書き:

- 料理名
- 食材
- 推定量
- カロリー
- たんぱく質
- 脂質
- 炭水化物

画像だけで量推定は難しいため、AI下書きからユーザー補正する流れを必須にする。

AI出力例:

```json
{
  "meal_name": "鶏むね肉定食",
  "items": [
    {
      "name": "ご飯",
      "estimated_amount": "150g",
      "calorie": 252,
      "protein": 3.8,
      "fat": 0.5,
      "carbs": 55.7
    },
    {
      "name": "鶏むね肉",
      "estimated_amount": "120g",
      "calorie": 130,
      "protein": 27.0,
      "fat": 2.0,
      "carbs": 0
    }
  ],
  "confidence": "medium",
  "needs_user_confirmation": true
}
```

UIでは「ご飯150gで合っていますか？」のように確認させる。

### 5.5 筋トレ管理

筋トレは計画と実績を分けて管理する。

計画:

- 日付
- 種目
- セット数
- 目標重量
- 目標回数
- 目標RPE
- メモ

実績:

- 実施日
- 種目
- セット数
- 実績重量
- 実績回数
- RPE
- 疲労度
- メモ

差分管理:

- 目標重量と実績重量の差分
- 目標回数と実績回数の差分
- 前回との比較
- 次回調整案

AIコメント例:

> 目標より2回少ないです。前回も同じ傾向なので、次回は重量を少し下げて10回達成を優先してもよさそうです。

### 5.6 AI改善提案

毎日または週次でAIが改善提案を出す。

入力に使うデータ:

- 体重推移
- 腹囲推移
- 食事記録
- 筋トレ記録
- 体型写真
- 睡眠
- 疲労度
- 目標

出力:

- 現在の状態
- 良い点
- 問題点
- 次の改善案

## 6. AI構成

### 6.1 基本方針

最初から完全自動にしない。AIは「下書き」と「提案」を担当する。

ユーザーが確認・修正できるUIを必ず入れる。

### 6.2 外部API案

MVPでは外部AI APIを第一候補にする。

理由:

- 実装が早い
- 精度が安定しやすい
- 比較検証しやすい

ただしコストがかかるため、初期ユーザー数が少ない間だけ使う。将来的にはローカルLLM化も検討する。

### 6.3 ローカルLLM案

ローカルで動かす場合は、Mac mini上にOllamaを置く。

候補:

- Gemma系
- Qwen系VLM
- その他画像対応VLM

処理:

1. iPhoneで写真撮影
2. サーバーまたはMac miniに送信
3. VLMで料理・体型を推定
4. JSONで返す
5. アプリ側で表示
6. ユーザーが補正

## 7. 栄養計算方針

画像AIにPFCを直接決めさせすぎない。

料理名と量を推定し、栄養DBで計算する。

必要なDB:

| テーブル | 主な項目 |
|---|---|
| FoodItem | name, calorie_per_100g, protein_per_100g, fat_per_100g, carbs_per_100g |
| Meal | date, meal_type, image_path, total_calorie, total_protein, total_fat, total_carbs |
| MealItem | meal_id, food_item_id, amount_g, calorie, protein, fat, carbs |

## 8. データモデル案

| モデル | 主な項目 |
|---|---|
| User | name, height, birth_year, sex, goal_type, target_weight, target_waist, activity_level |
| BodyLog | date, weight, waist, body_fat, chest, arm, thigh, calf, memo |
| BodyPhoto | date, angle, image_path, ai_comment, ai_score, memo |
| Meal | date, meal_type, image_path, total_calorie, total_protein, total_fat, total_carbs, ai_comment, confirmed_by_user |
| WorkoutPlan | date, menu_name, memo |
| WorkoutPlanSet | workout_plan_id, exercise_name, target_weight, target_reps, target_sets, target_rpe |
| WorkoutResult | date, menu_name, fatigue, memo |
| WorkoutResultSet | workout_result_id, exercise_name, actual_weight, actual_reps, set_number, rpe |
| AIInsight | date, insight_type, input_summary, output_comment, action_suggestion |

## 9. MVP再定義

最初のMVPは重くしすぎない。

### MVPに入れる

3. 体重・腹囲記録

5. 食事写真登録

## 11. 課金方針

初期は数人に完全無料で使ってもらう。目的は収益ではなく、継続利用とAI提案の価値検証である。

見ること:

- 毎日使うか
- 毎週使うか
- 食事写真を撮るか
- 体型写真を撮るか
- AI提案を読むか
- AI提案で行動が変わるか
- どこで面倒になるか

将来的な課金案:

| プラン | 内容 |
|---|---|
| 無料版 | 手動記録、体重記録、腹囲記録、筋トレ記録、食事の手動入力 |
| 有料版 月100円 | AI食事推定、AI体型写真比較、AI週次レポート、筋トレ差分コメント、改善提案 |
| 上位版 月500円から1,000円 | 無制限AI解析、本格減量コーチ、競技者モード、詳細レポート、パーソナル改善提案 |

## 12. 検証KPI

初期は売上ではなく、利用継続を見る。

見る指標:

- 初回登録者数
- 7日後継続率
- 30日後継続率
- 1人あたり記録回数
- 食事写真登録回数
- 体型写真登録回数
- 筋トレ記録回数
- AI提案の閲覧率
- AI提案後の行動率
- 手動補正率

特に重要:

- 食事写真を続けて撮るか
- 体型写真比較に価値を感じるか
- 筋トレ差分管理が便利か
- AIコメントが刺さるか

## 13. 実装フェーズ

| Phase | 目的 | 内容 |
|---|---|---|
| Phase 1 | 記録アプリとして成立させる | ユーザー、目的選択、体重、腹囲、食事写真、筋トレ記録 |
| Phase 2 | 差分管理を入れる | 筋トレ計画、筋トレ実績、目標との差分、週次グラフ |
| Phase 3 | AI下書きを入れる | 食事写真から料理推定、PFC下書き、ユーザー補正 |
| Phase 4 | 体型写真AIを入れる | 写真比較、見た目変化コメント、前回との差分 |
| Phase 5 | AIコーチ化する | 週次レポート、停滞原因推定、次週の改善案、目的別アドバイス |

## 14. 安全・表現方針

### 医療アプリにしない

診断、治療、病気判定はしない。

避ける表現:

- あなたは糖尿病です
- 病気です
- 医学的に危険です
- この食事で治ります

使う表現:

- 参考値です
- 傾向として
- 健康面で不安があれば専門家に相談してください
- 無理な減量は避けましょう

### 体脂肪率を断定しない

写真だけでは正確に分からない。

表現は以下にする。

- 推定
- 参考
- 見た目の変化
- 前回との差

### AIを過信させない

食事量は外れやすい。必ずユーザー補正できるようにする。

## 15. 最初に作るプロトタイプ

まずは以下だけ作る。

1. 体重・腹囲を入力できる
2. 食事写真を登録できる
3. 体型写真を登録できる
4. 筋トレ計画を登録できる
5. 筋トレ実績を登録できる
6. 計画と実績の差分を表示できる
7. AIが週次コメントを出す

最初のゴールは、数人が1週間使い続けるかを見ること。

完成度より、記録のしやすさとAIコメントの納得感を優先する。
