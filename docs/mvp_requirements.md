# MVP要件整理

作成日: 2026-07-01  
対象: iPhoneアプリ MVP  
参照資料: `gym_training_app_user_stories_mvp.md`, `docs/gym_training_app_design_mvp.md`, `docs/alpha_scope.md`

## 1. MVPの目的

MVPでは、ジムでのトレーニングを「計画する」「記録する」「振り返る」まで一連の流れとして成立させる。

特に重視する価値は以下。

1. ジム前に、種目・セット数・目標重量・目標回数を決められる
2. ジム中に、計画から迷わず記録を開始できる
3. セットごとに実績重量・実績回数・完了状態を残せる
4. 目標との差分、達成状態、総ボリュームを確認できる
5. 過去の履歴を見返して次回の参考にできる
6. オフラインでも記録が失われない

身体KPI、クラウド同期、分析グラフは重要だが、最初のMVP Coreでは中核体験の後に段階投入する。

## 3. MVP Coreの受け入れ条件

MVP Coreは以下を満たせば完了とする。

- プリセット種目から計画を作成できる
- 計画に複数種目を追加できる
- 種目ごとに複数セットを設定できる
- 各セットに目標重量・目標回数を設定できる
- アプリ再起動後も計画が残る
- 計画を選んでワークアウトを開始できる
- セットごとに実績重量・実績回数を変更できる
- セット完了チェックを付けられる
- 記録中にセット追加・削除、種目追加、種目スキップができる
- 完了画面で総ボリューム、種目達成率、全体達成率を確認できる
- 完了したワークアウトが履歴に保存される
- 履歴詳細でセットごとの実績を確認できる
- 主要フローのUIテストが通る

## 6. MVP対象外

初回MVPでは実装しない。設計上の拡張余地のみ残す。

| 領域 | 対象外 |
|---|---|
| 種目 | 別名統合、お気に入り、非表示、動画・画像 |
| 計画 | テンプレート、周期プログラム、代替種目提案 |
| 記録 | RPE/RIR、自重補助、時間・距離種目、スーパーセット |
| 履歴 | カレンダー表示 |
| 通知 | ローカル通知による休憩終了通知 |
| 連携 | Apple Health、Apple Watch、Widget、Live Activities |
| 同期 | CloudKit同期、競合解決、アカウント管理 |
| AI | メニュー提案、停滞検知、次回重量提案 |
| 共有 | SNS共有、友人フォロー |
| 課金 | 有料プラン、買い切り、サブスクリプション |
