# Apple Watchセンサー活用 ユーザーストーリー

更新日: 2026-07-18
対象: iPhone / Apple Watch版 AIボディメイクマネージャー

## 1. プロダクト原則

- センサーは入力を減らし、判断材料を増やすために使う。
- センサー値だけで医療判断、フォーム断定、体脂肪率断定をしない。
- 自動推定には信頼度を表示し、重量・回数・RPE・推定値を手動修正できるようにする。
- HealthKit、モーション、位置情報を許可しなくても、手動の筋トレ記録を完了できるようにする。
- 健康データをAIへ渡す設定は初期値をOFFにし、ユーザーの明示操作で有効にする。
- 高頻度センサー取得はワークアウト中・セット中に限定し、省電力設定を用意する。

## 2. 実装済みユーザーストーリー

### A. ジム前: 今日の判断

| ID | ユーザーストーリー | 利用データ | 受け入れ条件 |
|---|---|---|---|
| A-01 | 利用者として、今日行うメニューをiPhoneで選び、Watchですぐ開始したい | 計画、WatchConnectivity | 選択した計画がWatchの優先メニューとして表示される |
| A-02 | 利用者として、睡眠・安静時心拍・HRV・手首皮膚温の傾向から今日の回復状態を確認したい | HealthKit | 欠測値を0扱いせず、根拠とデータ充足度を表示する |
| A-03 | 利用者として、疲労が高い日は負荷を抑える提案を受けたい | 回復状態、目標、直近実績 | 医療判断ではない短い行動提案を端末内で表示する |
| A-04 | 利用者として、許可しない健康項目があってもアプリを使いたい | 権限状態 | HealthKit未許可でも計画・記録・履歴が動作する |

### B. ジム到着と開始

| ID | ユーザーストーリー | 利用データ | 受け入れ条件 |
|---|---|---|---|
| B-01 | 利用者として、登録したジムへの到着・退出をカレンダーに残したい | Core Location | ジオフェンスまたは手動チェックインで訪問日時を保存する |
| B-02 | 利用者として、位置情報を常時許可しなくてもジム訪問を記録したい | 手動入力 | 手動チェックイン・チェックアウトが利用できる |
| B-03 | 利用者として、筋トレ開始と同時にAppleのワークアウト計測も始めたい | HKWorkoutSession | 本アプリから筋力トレーニングを開始し、失敗時もセット記録を継続する |
| B-04 | 利用者として、権限が拒否されている理由を理解したい | HealthKit状態 | 許可状態と手動記録フォールバックを画面で説明する |

### C. セット中: ハンズフリー補助

| ID | ユーザーストーリー | 利用データ | 受け入れ条件 |
|---|---|---|---|
| C-01 | 利用者として、セット中も現在心拍・経過時間・消費エネルギーを見たい | HealthKit live builder | Watchの実行画面にライブ値を表示する |
| C-02 | 利用者として、手首の動きから回数候補を自動入力してほしい | Core Motion | セット中だけ推定し、信頼度と推定回数を表示する |
| C-03 | 利用者として、推定回数が違うときにすぐ直したい | 推定値、Digital Crown | 推定値の採用は任意で、リールまたは手入力で修正できる |
| C-04 | 利用者として、目標回数に達したことを画面を見ずに知りたい | 推定回数、目標回数 | 目標到達時に一度だけ触覚通知する |
| C-05 | 利用者として、動きが止まったらセット終了候補を出してほしい | モーション無活動時間 | 一定時間静止後に確認候補を表示し、自動確定はしない |
| C-06 | 利用者として、各セットの心拍と動作の安定性を後から確認したい | 心拍、反復間隔、振幅 | 平均・最大心拍、推定回数、平均反復時間、一貫性をセットに保存する |
| C-07 | 利用者として、体感強度も残したい | RPE | RPE 6〜10または未設定をセット単位で保存する |

### D. セット間: 回復と休憩

| ID | ユーザーストーリー | 利用データ | 受け入れ条件 |
|---|---|---|---|
| D-01 | 利用者として、セット終了後に休憩タイマーを自動開始したい | セット完了、計画休憩時間 | 5秒刻みの設定値から自動開始し、停止・スキップできる |
| D-02 | 利用者として、次セットへ進めそうか知りたい | 心拍低下、RPE、経過時間 | 心拍回復とRPEを使った参考メッセージを表示する |
| D-03 | 利用者として、きついセットの後は休憩時間を調整してほしい | RPE、心拍 | 設定ON時だけ休憩を延長し、手動変更を優先する |
| D-04 | 利用者として、休憩終了を見逃したくない | タイマー | 終了時に触覚通知する |

### E. ジム後: 保存と振り返り

| ID | ユーザーストーリー | 利用データ | 受け入れ条件 |
|---|---|---|---|
| E-01 | 利用者として、筋トレをApple Healthにも残したい | HealthKit | 時間・心拍・活動エネルギーをワークアウトとして保存する |
| E-02 | 利用者として、Health保存に失敗しても筋トレ記録を失いたくない | ローカル永続化 | アプリ内記録を先に保持し、Health保存状態を履歴に表示する |
| E-03 | 利用者として、セットごとの回数・重量・RPE・心拍をiPhoneで見たい | Watch同期 | 完了実績とセンサー要約をiPhone履歴へ重複なく同期する |
| E-04 | 利用者として、トレーニング中の気づきを音声入力したい | Watch文字入力・音声入力 | Watchのシステム入力でメモを追加し、履歴へ同期する |
| E-05 | 利用者として、当日と週単位の運動量を比較したい | 歩数、活動量、トレーニング量 | 日次コンディションと7日分析を表示する |
| E-06 | 利用者として、伸び悩みや負荷の偏りに気づきたい | 直近セッション | 週ボリューム、密度、達成率、停滞候補を端末内で算出する |
| E-07 | 利用者として、食事量と消費量の関係を見たい | 食事、活動・基礎消費 | 推定であることを明記し、摂取と消費の参考差分を表示する |

| ID | ユーザーストーリー | 受け入れ条件 |
|---|---|---|
| F-01 | 利用者として、使うセンサーを自分で選びたい | Health、モーション、適応休憩、触覚、位置情報を個別設定できる |
| F-02 | 利用者として、バッテリー消費を抑えたい | 省電力モーション設定を用意し、セット外では高頻度取得しない |

## 3. データ品質と推定ルール

| 項目 | ルール |
|---|---|
| 回数推定 | 片手首の加速度ピークから候補を出す。種目、装着腕、可動域により誤差があるため自動確定しない |
| テンポ | 有効な反復間隔の平均を参考値として保存する |
| 動作一貫性 | 反復間隔と振幅のばらつきから0〜1で表す。フォーム良否の断定には使わない |
| 回復状態 | 個人の14日ベースラインとの差を使い、欠測項目を評価対象から外す |
| エネルギー差 | 食事記録とHealthKitの推定消費量による参考値。減量結果を保証しない |
| 心拍回復 | セット終了後の短時間心拍低下を休憩判断の補助にする。診断には使わない |
| ジム滞在 | ジオフェンスはOS・電波・権限の影響を受けるため、手動訂正を可能にする |

## 4. OS・ハードウェア上の制約

- Apple Watchの世代や許可状態によって取得できるHealthKit項目は異なる。
- Apple Watch SEではECG、血中酸素、手首皮膚温など一部機能を利用できない場合がある。画面は常に欠測を許容する。
- 転倒検出、衝突事故検出のシステムイベントや、生の環境光値を一般アプリのトレーニング分析へ直接取り込む設計にはしない。
- 血圧・血糖をWatch単体で測定した値として扱わない。将来HealthKitに外部機器データがある場合だけ、別機能として検討する。
- GPSルートはランニング等の屋外ワークアウト向け機能とし、屋内筋トレMVPではジム訪問記録を優先する。

## 6. KPI

- Watch開始率: Watchへ送った計画のうち開始された割合
- Watch完了率: Watch開始セッションのうち終了保存された割合
- 自動回数採用率と手動修正率
- センサー付きセット率
- 休憩提案採用率
- Health保存成功率
- 7日後・30日後の記録継続率
- 週次分析の閲覧率と提案後行動率
